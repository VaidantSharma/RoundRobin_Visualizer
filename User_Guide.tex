\section{Setup Instructions}

\subsection{Prerequisites}
\begin{itemize}[leftmargin=*,itemsep=0pt]
    \item \textbf{Node.js}: Version 18.0.0 or higher
    \item \textbf{npm}: Version 9.0.0 or higher
    \item \textbf{Modern Web Browser}: Chrome 120+, Firefox 121+, Edge 120+, or Safari 17.0+
\end{itemize}

\subsection{Installation}
\begin{enumerate}[leftmargin=*,itemsep=0pt]
    \item Clone the repository: \texttt{git clone https://github.com/VaidantSharma/RoundRobin\_Visualizer.git}
    \item Navigate to project directory: \texttt{cd RoundRobin\_Visualizer}
    \item Install dependencies: \texttt{npm install}
    \item Start development server: \texttt{npm run dev}
    \item Open browser: \texttt{http://localhost:3000}
\end{enumerate}

\textbf{Alternative:} Access the live demo at \href{https://roundrobinvisualizer-woad.vercel.app/}{roundrobinvisualizer-woad.vercel.app} without installation.

\section{User Interface Guide}

\subsection{Process Configuration}
\begin{itemize}[leftmargin=*,itemsep=0pt]
    \item \textbf{Add Process}: Click "Add Process" to create new processes (default: burst=5, arrival=0)
    \item \textbf{Burst Time}: Total CPU time required (minimum: 1)
    \item \textbf{Arrival Time}: Time unit when process arrives (minimum: 0)
    \item \textbf{Remove}: Click trash icon to delete a process
    \item \textbf{Time Quantum}: Maximum CPU time per process before preemption (default: 4)
    \item \textbf{Speed}: Animation speed slider (1x to 100x)
\end{itemize}

\subsection{Controls}
\begin{itemize}[leftmargin=*,itemsep=0pt]
    \item \textbf{Start/Pause}: Toggle simulation execution
    \item \textbf{Reset}: Restore initial state and clear all data
    \item \textbf{Real-time Info}: Displays current time, context switches, executing process, quantum left, and remaining time
\end{itemize}

\subsection{Visualization Areas}
\begin{enumerate}[leftmargin=*,itemsep=0pt]
    \item \textbf{Process Configuration Panel}: Add/configure processes and system parameters
    \item \textbf{Control Panel}: Simulation controls and real-time statistics
    \item \textbf{Process Flow Animation}: Visual representation of process lifecycle
    \item \textbf{Gantt Chart}: Timeline view of process execution sequence
    \item \textbf{Performance Metrics}: Detailed statistics table with averages
\end{enumerate}

\section{Animation Features}

\subsection{Color Coding}
Each process has a unique color: Blue (P1), Green (P2), Orange (P3), Red (P4), Purple (P5), Pink (P6), Cyan (P7), Lime (P8). Colors are consistent across all visualizations.

\subsection{Visualization Flow}
The application uses a top-to-bottom flow:
\begin{enumerate}[leftmargin=*,itemsep=0pt]
    \item \textbf{New Arrival} (Purple theme): Processes appear at their arrival time
    \item \textbf{Ready Queue} (Blue theme): FIFO queue of processes waiting for CPU
    \item \textbf{CPU Execution} (Yellow theme): Currently executing process with pulsing animation
    \item \textbf{Completed} (Green theme): Finished processes with checkmark symbol
    \item \textbf{Return to Queue} (Orange): Preempted processes returning to ready queue
\end{enumerate}

\subsection{Animation Transitions}
\begin{itemize}[leftmargin=*,itemsep=0pt]
    \item \textbf{Slide In Top}: New processes arriving (purple border)
    \item \textbf{Move Down Row}: Moving to queue/CPU/completed sections
    \item \textbf{Pulse}: Continuous animation for executing process
    \item \textbf{Move Up To Queue}: Preempted process returning to queue (orange border)
\end{itemize}

\subsection{Performance Metrics}
The application calculates:
\begin{itemize}[leftmargin=*,itemsep=0pt]
    \item \textbf{Turnaround Time} = Completion Time - Arrival Time
    \item \textbf{Waiting Time} = Turnaround Time - Burst Time
    \item \textbf{Average Turnaround Time} and \textbf{Average Waiting Time}
    \item \textbf{Total Context Switches}
\end{itemize}

Metrics are displayed in a table with per-process details and aggregate summary cards.
